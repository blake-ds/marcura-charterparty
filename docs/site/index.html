04Strike-through removal

Negotiators physically strike out the boilerplate they reject; the replacement language sits next to it. Including struck text would silently change the contract. In this PDF, strikes are not a font feature — they are thin filled rectangles drawn over the glyphs (height ≈ 0.42 pt). A glyph is "struck" when its bbox centre lies inside one of those rectangles.

PDF — clause 6 line 118 Dues … loaded or discharged, and any taxes on freight whatsoever shall be paid by Owners …
→
clauses.json — shellvoy-6.text Dues … loaded or discharged, and shall be paid by Owners …

Whole-line vote

Drop a line only when its first printable glyph is struck and ≥ 50 % of the line is struck. Visible-first lines survive.

Word-level promotion

Any struck glyph inside a word → drop the whole word. Catches the orphan-suffix pattern where a strike rectangle ends mid-word. PDF: [struck: in]demnity follows parser: word "indemnity" dropped JSON: "… follows …"

Sandwich rule

A clean-ish line tightly between two heavy-struck lines is dropped — surviving glyphs are debris from a partial strike rect, not real content.

Punctuation-led exception

If the visible tail of a mostly-struck line opens with , or :, keep it. A clean continuation is still content.

06LLM as verifier, never transcriber

The deterministic parser writes every byte of clauses.json. The Azure-hosted LLM only reviews — flags suspicious boundaries (truncated, merged, out-of-order) as advisory warnings.

RoleDefault deploymentEndpointInput shape
VerifierDeepSeek-V4-FlashAzure FoundryHTML-tagged clauses, strict-JSON response

Switch deployments via MARCURA_VERIFIER_MODEL in .env; deployment names prefixed deepseek, grok, mistral, llama, or cohere are routed through Azure Foundry, the rest through Azure OpenAI. A verifier outage never breaks a deterministic extraction — the runner logs anomalies and returns normally.

Current advisory output — three known false positives (shellvoy-4, additional-1, additional-11) where the deterministic parser correctly ends mid-sentence because the rest of the clause is struck out on the page. Each is documented and cross-referenced to eval/ground_truth.md in architecture.md; a new flag would warrant a manual page-by-page check.

07Evaluation

Deterministic, no LLM. make eval runs the same gate CI runs — every check green on every commit.

25 of 25 checks pass against a hand-curated golden file: counts, exact ordinal lists, frozen text on regression-prone clauses, "struck must be absent", "kept must be present", and "no embedded next-anchor leakage" — backed by a page-by-page manual ground truth at eval/ground_truth.md. 25 / 25

08Trade-offs

Determinism > cleverness
Every byte of text is byte-for-byte traceable to a glyph on the source page. A vision-only LLM pipeline would have been faster to write, riskier to ship.
Two parsers, not one
Two-column SHELLVOY and inline rider sections share zero geometry. Forcing a unified parser would have hurt readability with no fidelity gain.
Verifier is advisory
The LLM must not author text in a fidelity-critical legal pipeline. It judges what the parser produced; logs warnings; never raises.
Pydantic everywhere
One model story end-to-end (parser internals → wire JSON). No @dataclass; no parallel serialization paths.
Faithful, not opinionated
Visible glyphs go to JSON. Wholly-replaced clauses — title struck and body reduced to mid-word fragments — are dropped (21 in this corpus, enumerated in eval/ground_truth.md).
Markdown source, HTML showcase
The docs/*.md are the canonical source — agent-friendly, GitHub-friendly, diff-friendly. This page is a hand-crafted snapshot; no framework, single file.
